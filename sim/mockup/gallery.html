Earlier explorations — the four disclosure directions

1 · Board + callouts whole system always visible

The entire schematic on one screen. Tap a part and a bubble pops up anchored right at it, expanding from a peek to its controls; everything else dims.

Best overview density · risk: tight on a phone, callouts can crowd.

2 · Accordion most compact

A scannable list of collapsed zone strips (state pill + flow total). Tap one to expand its mini-schematic and controls inline; others stay collapsed.

Best for scanning state fast · risk: loses the always-on whole-system picture.

3 · Map + detail pane master / detail

A small fixed map of the whole system up top stays put; the bottom pane swaps to show the tapped component large with its values + controls.

Overview never moves while you work · risk: map gets small, detail is half a screen away.

4 · Semantic zoom one canvas

One canvas where detail is a function of zoom: system → zone → component. Zoom out = clean overview; zoom in = labels, values, then in-place controls.

Most “native app” feel, no separate panels · risk: more interaction to learn, zoom can disorient.

Our system drawn in reference styles

Same pipes & wiring, three real visual languages — react to the feel.

A · Circuit-sim Falstad style

Spare technical schematic on a grid; moving dots animate along both pipes and energized wires; click a part for an inline editor. The closest analog to what we're actually building.

Best for the live-flow feel · look is bare/engineering.

B · SCADA / P&ID operator HMI

Industrial control screen with standard symbols (impeller pump, bowtie valves, sprinkler glyphs), orthogonal pipes, state by colour, alarm banner; tap a device for a faceplate.

Most “real instrument” & recognizable symbols · can feel utilitarian.

C · Node-RED flow editor

Every item is a node with typed ports (water / signal / power) joined by bezier wires that animate when live; status dot per node; tap a node for its config drawer.

Cleanest “controls live in their items” · less like a physical schematic.
